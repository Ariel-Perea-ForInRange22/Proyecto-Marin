# Pruebas del Proyecto Marin

Este directorio concentra la estrategia de pruebas unitarias y su documentacion.

## Objetivo

Asegurar que los componentes criticos del proyecto funcionen correctamente y que cada prueba deje evidencia clara de:

- Por que se hizo la prueba
- Que se esperaba
- Como reacciono la aplicacion
- Resultado final

## Estructura sugerida

- PLAN_PRUEBAS.md: alcance, prioridades y criterios de salida
- MATRIZ_CASOS_INICIALES.md: lista base de casos de prueba propuestos
- PLANTILLA_CASO_PRUEBA.md: formato estandar para documentar cada prueba
- evidencias/: capturas, logs, salidas de consola

## Flujo recomendado

1. Seleccionar un caso de la matriz.
2. Copiar la plantilla y crear un archivo por prueba.
3. Ejecutar la prueba y guardar evidencia.
4. Documentar resultado y comportamiento observado.
5. Registrar acciones (si falla: bug, fix, re-test).
